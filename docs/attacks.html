Domain model

What AttackAggregate models

Each attack definition captures three things: a unique name that becomes the task title seed, an attack category drawn from the AttackCategory enum (12 values), and any domain-specific context fields declared in the DSL.

AttackAggregate is a pure value object — no persistence, no side effects. It is instantiated by the ANTLRParserAdapter when it walks the ANTLR parse tree for a given .attack file.

The FilesystemAttackRepository implementation calls parseDirectory() on the ANTLR adapter and returns a flat AttackAggregate[] array. Everything downstream — the scheduler, the task generator — receives only this array.

Source format

The .attack DSL

Each file in models/ describes exactly one attack. The grammar is defined in grammar/AttackDef.g4 and compiled by ANTLR 4 into a PHP parser.

models/phishing-email.attack (conceptual)
attack PhishingEmail {
    category  SOCIAL_ENGINEERING
    severity  HIGH
    surface   email
    technique credential-harvesting
}
Parsing entry point — bin/parse-attacks.php
$parser  = new ANTLRParserAdapter();
$attacks = $parser->parseDirectory($projectRoot . '/models');
// $attacks is AttackAggregate[]

Live rendering

Browse the interactive catalog

The static site at /catalog/ is generated at CI time by bin/build-static-site.php, which feeds the parsed AttackAggregate[] array into AttackListRenderer. The result is a single self-contained HTML file — CSS-only modals, dark theme, PWA manifest, no JavaScript framework required.
